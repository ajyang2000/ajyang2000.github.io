Designing a Next-Generation Frontend

Polaris Intelligence is an online business intelligence application that provides users with consumer data and analytical insights via their online applicaiton. The current application is managed using PHP and JavaScript. However, PHP is currently deprecated, which causes the interface to appear simple and plain whereas JavaScript has limited capabilities, such as the ability to not easily implement drag and drop functionality. As a result, our client, Polaris Intelligence, is seeking a next-generation front end application. For this capstone project, done in collaboration with three other students in differing diciplines, aimed to improve on the current, outdated interface of Polaris Intelligence's online application using Node.js and React.

After understanding the problem better via meetings with the clients, the team analyzed the existing design of the interface, comparing the application to Nielsen's 10 Usability Heuristics for User Interface Design. From the analysis, the team were able to generate three different low fidelity prototypes, where each prototype focused greatly on one of:

  1. User interactions (For instance, hovering over buttons)
  2. Ease of navigation to common pages of the application
  3. Improving client's requirements

The low fidelity prototypes were then analyzed and used in creating a final proposed candidate design, which includes a wireframe of the prototype as well as a digital visualization of the prototype. The proposed candidate design was then used as a stepping stone for the high fidelity prototype.

Experience

For the project, I was the primary member responsible for communications in the team. As such, I was tasked with scheduling the necessary meetings with the client and our supervisor, as well as personally contacting the team members in case of a problem. Additionally, any deliverables sent to the supervisor and inquiries for the state of the project were handled by me. Apart from communications, I have contributed to designing the project requirements and design proposal documents and creating an alternative design for the current application as of this date.

This team project showcased the skills learnt during my academic career. The following skills were utilized in creating an application for the client in a progressive format.
  • HTML, CSS, JavaScript - used in designing the final prototype
  • Collaboration - contributed the knowledge gained from my discipline towards the final project, as well as incorporating the knowledge with the other disciplines
  • Time Management - ensured that the internal deadlines were met and considered for unexpected scenarios
Apart from the existing skills, the project also developed the following, which was useful in solving this problem:
  • Node.js and React
  • External Communications, specifically with clients
  • Project Management Skills

Teafenders

Link to itch.io page: Teafenders by Saucer Games

The project required for this course involves creating a 3D game with various students in the University of Toronto Computer Science, University of Toronto Music, Centennial College Game Art and Centennial College Animation. For this project, the team, comprised of a total of 12 students and myself, created a casual cooperative game, called Teafenders. Teafenders is a short defense game where 1-4 players protect teacups from living teabags for a given time. Teamwork is a core element and is best suited for a group of players. Throughout the period of 10 weeks, the beta version of the game was developed, incorporating any feedback given by the students, professors or any well established studios such as Ubisoft Toronto, Snowman and Zynga.

Experience

This team project showcased mainly my programming knowledge. During the pre-alpha state of the project, I initially contributed to the mechanics (movement, jump and attacking) and implementing the animations (walk) of the players’ character, utilizing my programming skills in Unity and C#. From the alpha version onwards, my main contribution was on the UI aspects of the game, both in the menus and in-game, as well as creating the save system that allows the scores gained to be retained. Some of the menu and in-game tasks include:

  • Completing the button functions for each menu
  • Loading the appropriate images when the character and level are selected. Also, I needed to ensure that the correct data is sent to the UI during gameplay.
  • Implementing the UI on each character’s models
  • Scaling the UI to adjust to screen size
  • Creating a save and load system for the scores
  • Creating the main, controls, levels, and end scenes for the game.

Abandon the Spaceship

Link to itch.io page: Abandon The Spaceship

Abandon the Spaceship is a game where two players compete to escape a damaged spaceship. The players must get the key to unlock the door in puzzle rooms, while using traps to stall the opposing player. The game was created as an entry for the UTGDDC Game Making Deathmatch 2020/21 .

Experience
